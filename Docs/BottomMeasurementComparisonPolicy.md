# 하의 실측 비교 정책

## 측정 코드

하의 실측은 이름이 같아도 시작점이 다르면 별도 코드로 저장한다.

| 화면 항목 | 측정 코드 | 정의 |
| --- | --- | --- |
| 허리단면 | `waist_width_edge_to_edge` | 허리단 양쪽 끝 사이 |
| 엉덩이단면 | `hip_width_at_widest` | 엉덩이의 가장 넓은 지점 양쪽 끝 사이 |
| 허벅지단면 | `thigh_width_crotch_to_outer` | 가랑이 봉제점 바로 아래부터 바깥쪽 끝까지 |
| 밑위 | `rise_crotch_to_waist_front` | 앞 가랑이 봉제점부터 허리단 위까지 |
| 밑단단면 | `hem_width_edge_to_edge` | 한쪽 바짓단 양쪽 끝 사이 |
| 총장(아웃심) | `pants_outseam_waist_to_hem` | 허리단 위쪽부터 바깥 봉제선을 따라 밑단까지 |
| 인심 | `pants_inseam_crotch_to_hem` | 가랑이점부터 안쪽 봉제선을 따라 밑단까지 |

아웃심과 인심은 모두 길이 값이지만 직접 호환하지 않는다. 직접 측정의 `총장`은 아웃심 코드로 저장한다.

## 확정 추천 최소 기준

- 동일한 측정 코드가 최소 2개 필요하다.
- 그중 최소 2개는 `허리단면`, `엉덩이단면`, `허벅지단면`이어야 한다.
- 밑위, 밑단, 총장은 보조 근거로 점수에 반영하지만 핵심 폭 항목을 대신할 수 없다.
- 긴바지와 반바지를 직접 비교하면 총장은 제외한다.

예를 들어 `허리단면 + 총장`만 호환되면 수치는 보여줄 수 있지만 추천을 확정하지 않는다. `허리단면 + 엉덩이단면`이 호환되면 최소 기준을 충족한다.

## 쇼핑몰 데이터 처리

쇼핑몰 원본 필드가 `inseam`처럼 측정 정의를 명시한 경우에만 해당 코드로 매핑한다. `기장`, `총장`, `length`처럼 시작점이 불명확한 필드는 원본값을 보존하되 호환 코드로 추정하지 않는다.
