# 아우터 실측 비교 정책

## 비교 항목

아우터는 다음 측정 항목을 사용한다.

| 항목 | 측정 코드 | 역할 |
| --- | --- | --- |
| 가슴단면 | `chest_width_pit_to_pit` | 필수 핵심 항목 |
| 어깨너비 | `shoulder_width_seam_to_seam` | 핵심 보조 항목 |
| 총장 | `body_length_hps_to_hem_front` | 길이 보조 항목 |
| 소매길이 | `sleeve_shoulder_seam_to_cuff` 등 | 구조가 호환될 때 사용하는 보조 항목 |
| 밑단단면 | `hem_width_edge_to_edge` | 실루엣 보조 항목 |

측정 코드가 같은 값끼리만 비교한다. 목 중심 기준 소매, 어깨 봉제선 기준 소매, 라글란 소매는 각각 별도 코드로 유지한다.

## 확정 추천 최소 기준

- 동일한 측정 코드가 최소 2개 필요하다.
- 가슴단면은 반드시 포함돼야 한다.
- 어깨, 총장, 소매, 밑단은 두 번째 이후의 비교 근거로 사용할 수 있다.
- 가슴단면 없이 어깨와 소매만 일치하면 수치는 참고할 수 있지만 추천을 확정하지 않는다.
- 봉제 구조가 명확히 다르면 자동 기준 옷 후보에서 제외한다.

## 직접 측정

아우터 밑단단면은 밑단 조절끈을 당기지 않은 상태에서 양쪽 끝을 수평으로 측정한다. 직접 측정값에는 `fitmatch_standard_v1`을 저장한다.

## 쇼핑몰 데이터 처리

쇼핑몰 원본값은 모두 보존한다. `가슴`, `총장`, `밑단`처럼 이름만 있고 측정 시작점이나 위치가 불명확한 값은 호환 코드로 추정하지 않는다.
